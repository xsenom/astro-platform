# Структура маршрутов (`src/app`)

Проект использует **Next.js App Router**. В этой модели URL строится по папкам, а не по именам отдельных файлов.

## Почему под каждую страницу отдельная папка

В App Router каждая папка — это сегмент маршрута. Внутри сегмента можно хранить:

- `page.tsx` — контент страницы;
- `layout.tsx` — общий layout для группы страниц;
- `loading.tsx`, `error.tsx`, `not-found.tsx` — служебные состояния;
- вложенные сегменты и API-роуты.

Поэтому «все страницы в одной папке разными файлами» — это подход старого `pages`-роутера, но не `app`-роутера.

## Как теперь сгруппировано (по ответственности)

Чтобы было проще ориентироваться, маршруты разложены по route groups (папки в скобках):

- `(marketing)` — публичные страницы (`/`, `/main`, `/guide/uran-v-bliznetsah`, `/uran-v-bliznetsah`);
- `(auth)` — авторизация (`/login`, `/reset-password`);
- `(account)` — личный кабинет (`/cabinet/*`);
- `(backend)` — API-роуты (`/api/*`).

> Папки в скобках **не попадают в URL**. Они нужны только для структуры кода.

## Важно

- Да, после такой группировки всё продолжит работать, если сохраняются названия route-сегментов (`cabinet`, `api`, `main` и т.д.) внутри групп.
- Переименовывать стоит не URL-сегменты, а **группы и внутреннюю организацию**, чтобы не ломать внешние ссылки.
